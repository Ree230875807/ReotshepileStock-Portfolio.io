## Reflection on markdown

Markdown helped me learn how developers write professional project documentation.
It helped me organize my portfolio clearly with headings, lists, and links.

Markdown made my repository look better and more professional.
I learned that writing good code is just as important as writing good documentation.

## Reflection on the Mock Interview

The mock interview helped me feel more sure of myself and better at talking to people.
It let me talk about my projects in a professional way.
I understood how important it was to get ready to answer questions about my technical skills and how well I work with others.
The interview taught me how to better present myself to future employers.

## Reflection on Deploying GitHub Pages

Using GitHub Pages to deploy my portfolio taught me how to publish a live website straight from a repository.
I learned how hosting and version control work together.
This experience helped me learn more about deploying websites and giving professional presentations online.
It also taught me how important it is to test my website after I put it online.
